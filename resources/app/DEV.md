# Инструкции по разработке

## Проблема с ES6 модулями и file:// протоколом

Если вы открываете `index.html` напрямую в браузере (через `file://`), ES6 модули не будут работать из-за политики CORS браузера. Это нормальное поведение безопасности браузера.

## Решения

### Вариант 1: Использовать Electron (рекомендуется)
```bash
npm start
```
Это запустит приложение в Electron, где модули работают нормально.

### Вариант 2: Использовать локальный HTTP сервер

1. Установите зависимости:
```bash
npm install
```

2. Запустите локальный сервер:
```bash
npm run dev
```

Это откроет приложение в браузере на `http://localhost:8080`

### Вариант 3: Использовать Python (если установлен)

```bash
# Python 3
python -m http.server 8080

# Python 2
python -m SimpleHTTPServer 8080
```

Затем откройте в браузере: `http://localhost:8080`

### Вариант 4: Использовать другие инструменты

- **PHP**: `php -S localhost:8080`
- **Node.js http-server**: `npx http-server . -p 8080`
- **VS Code**: Расширение "Live Server"

## Важно

- **Electron** работает нормально, потому что у него свой контекст выполнения
- **Браузер** требует HTTP сервер для ES6 модулей
- Это нормальное поведение безопасности браузера, не баг
